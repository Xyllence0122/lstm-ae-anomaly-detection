# V3.2 稽核修正說明

## 版本策略

- `final_holdout_v3.json`、`final_holdout_v3_1.json` 與 `external_real_data_v3_1.json` 保持 byte-for-byte 不變。
- `report_errata_v3_family.json` 綁定上述報告 SHA-256，撤回或修正不正確的欄位與說法。
- V3.2 使用版本化資料、artifact、calibration report 與全新 holdout seeds，不覆寫 V3/V3.1。
- V3.2 模型架構未變，重跑後的 `state_dict` 與 V3 逐 tensor 相同；threshold 因新的 normal-only calibration cohort 而版本化改變。

## 逐項修正

1. Type A support：`onset_index` 與 `end_index` 現在由注入前生成基線和注入後波形的逐點差異計算，定義為 pre-quantization intervention 改變基線超過 `1e-12` 的第一與最後樣本。seeds 10–19 regression test 及人工 support test 已加入。用原 selection seeds 重建時，所有波形逐點不變，300 筆 A 類 end metadata 改正，例如 12 改為 98。
2. Threshold provenance：V3.2 locked report 明確記錄 seed 396001、10,000 normal、calibration report SHA-256、`23_calibrate_v3_2.py` SHA-256 及來源 artifact SHA-256。
3. 真實資料歷史重用：43 normal + 20 faulty 與 64/43 split 均已在 V2 使用。新報告只稱為「未用於 V3 權重/threshold 的歷史重用資料稽核」，不稱 unseen、untouched 或獨立外部驗證。
4. Timing feature：程式沒有 timestamp-aware rate，也沒有固定 1 Hz 重採樣。輸入是 `x[t]-x[t-1]` 每樣本差分與 `sample_index / expected_samples` 樣本進度。
5. V3 status：第一次 V3 locked holdout 仍是其 frozen artifact 的一次性評估，但其 FPR 結果促成 V3.1 protocol，不能代表整個 V3 family 的 no-further-tuning。erratum 已明確標記。
6. Training provenance：V3.2 corrected selection report 保存完整 argv、data path/hash、3 seeds、40 epochs、batch size 128、samples-per-size 4、dynamic weight 0.5、固定視窗/分數設定，以及每 seed `completed_epochs=40` 和 checkpoints `[10,20,30,40]`。
7. Pre-onset diagnosis：profile 先從 normal event scores 決定 `base_scale`，再用該 threshold 呼叫 `event_decisions` 計算 pre-onset crossing；不再以 `np.inf` 產生恒 false 診斷。已加入 onset 前超標、onset 後未偵測的測試。
8. Edge 狀態：V3.2 仍只有 `.pt` 與離線多尺度 evaluator；沒有 V3 TorchScript、串流 runtime 或 Raspberry Pi 5 benchmark，不能稱 edge-deployment ready。

# V3 模型與實驗說明

## 研究題目

基於邊緣運算與時間序列模型之半導體製程即時異常偵測系統之架構

## V2 問題與 V3 修正

V2 表現不佳不只是模型容量問題。資料稽核確認下列設計缺陷：

1. 106/107 片正常晶圓的原始陣列在尾端出現時間倒序。V2 未先依 `Time` 排序，導致模型把人為的尾端跳變學成正常波形。
2. 建模用正常晶圓包含兩個明顯壓力操作群，平均壓力約為 1189 與 1232。V2 以單一平均與高斯偏移描述兩群，產生了原始資料中不存在的中間操作狀態。
3. V2 各 sensor 獨立產生雜訊，未保留跨 sensor 共變異。
4. 舊 A 類只把平均波形加速播放，但實際輸出不一定超出正常變化範圍，因此會出現「標籤是異常、觀測卻與正常不可分」的情況。
5. LAM 9600 資料取樣週期約為 1 秒。0.1 秒或 0.001 秒內已完成且未留下後續影響的事件，無法由這份資料直接證明可偵測。

V3 的資料處理依序執行：篩選 step 4/5、依時間穩定排序、相同時間戳取平均、確認時間嚴格遞增。正常資料只使用建模組估計統計量，再依壓力平均值最大間隔分成兩個操作群；每群分別估計時間對齊後的正常波形、片間偏移、跨 sensor 殘差共變異、lag-1 自相關及量化階距。

## 異常定義

V3 異常強度在模型選擇前固定，沒有依 final holdout 結果修改：

- A 類動態變化率異常：任一至兩個 sensor 的實際變化率必須達到所屬操作群正常 99 百分位的 1.25 至 1.75 倍，並包含 3 至 6 倍快速暫態。
- B 類製程震盪：任一至兩個 sensor 注入正常殘差標準差 4 至 7 倍的有限時間震盪。
- C 類漸進漂移：任一至兩個 sensor 從製程中段開始，最終偏移為正常殘差標準差 4 至 6 倍。

這些條件代表合成的中高強度異常，不代表已涵蓋所有真實機台故障。

## 模型與即時性

主模型仍是 Sliding-Window LSTM Autoencoder，並未改成 Isolation Forest 或 LSTM Forecaster。V3 的差異如下：

- 直接用長度 8、16、32、64 的滑動視窗訓練，不再以完整 cycle 權重假設等同視窗模型。
- 輸入包含 4 個 sensor 數值、4 個一階差分及 1 個經過樣本比例，共 9 個因果特徵。
- 訓練損失同時約束數值重建與一階差分重建。
- 同一組 LSTM-AE 權重建立多時間尺度分數；短視窗抓快速局部變化，長視窗抓震盪與漂移。
- 所有尺度的最大正規化分數共同校準總 FPR，並非把多個各自 0.1% 的誤報率直接相加。
- 任一時刻的特徵與視窗只使用當下及過去資料，不使用未來樣本。

目前模型假設資料先被重採樣為約 1 Hz。未來 Pi 5 串流端必須檢查 timestamp 與取樣週期；若改成 10 Hz 或更高頻率，必須用相同頻率重新建立正常分布與訓練模型。

## 驗證集結果

正式訓練使用 3 個 seed、每個 40 epoch，並保留每 10 epoch checkpoint。三個 seed 的單尺度 macro recall 為 71.0%、68.1%、71.6%，release checkpoint 是 seed 34003、epoch 10。

研究模式在 2,000 片正常與每類 300 片合成異常的選擇資料上：

- FPR：0.5%
- Precision：98.64%
- Recall：80.33%
- F1：88.55%
- A/B/C recall：66.67% / 89.33% / 85.00%

低誤報模式在相同選擇資料上：

- FPR：0.1%
- Precision：99.70%
- Recall：73.56%
- F1：84.65%
- A/B/C recall：56.00% / 83.33% / 81.33%

Precision 99.70% 不能解讀成「產線上 99.70% 的警報都正確」，因為此資料的異常比例遠高於真實產線。若真實異常率為 0.1%，依目前驗證 recall 與 FPR 推估，警報 Precision 約為 42.4%。因此 V3 仍是論文原型，尚未達到量產部署標準。

## 實驗隔離

- 合成訓練與選擇資料 seeds：31001、32001、33001。
- 模型訓練 seeds：34001、34002、34003。
- 模型、視窗、分數模式及閾值只用選擇資料決定。
- locked final holdout 使用全新 seeds 294001、295001。
- final holdout 報告不可覆寫；若模型、統計檔或程式變更，必須更換 artifact、seeds 與報告檔。

## 第一次 locked holdout 結果

在模型與 threshold 固定後，使用 seed 294001 產生 10,000 片正常資料，並以 seed 295001 產生每類 1,000 片異常。結果如下：

- TP / FN / FP / TN：2266 / 734 / 32 / 9968
- Accuracy：94.11%
- Precision：98.61%
- Recall：75.53%
- F1：85.54%
- A/B/C recall：58.60% / 82.70% / 85.30%
- FPR：0.32%，95% Wilson interval 為 0.227% 至 0.451%

這個 FPR 明顯高於選擇資料設定的 0.1%，表示 2,000 片正常驗證資料只容許約 2 個 false positive，無法穩定估計低尾端 threshold。不得用本次 holdout 的 32 個 false positive 重新選 threshold。後續 V3.1 只可使用全新且不含異常的 10,000 片正常 calibration cohort 校準操作點，並使用另一組新 seeds 與新報告檔複驗。第一次報告 `final_holdout_v3.json` 必須永久保留。
